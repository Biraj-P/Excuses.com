Excuses.com

Deployment Guide

⚠️ Important: This application requires a server-side component or serverless functions to work with the Together API. GitHub Pages does not support server-side code and will only work with the local excuse database fallback.

Deployment Options

1. Netlify (Recommended)

1 Sign up for a Netlify account
2 Connect your GitHub repository or upload this project
3 Set up environment variables in Netlify: TOGETHER_API_KEY=your_api_key_here

The serverless function in netlify/functions/together-proxy.js is already set up to handle API calls securely.

2. Vercel

1 Sign up for a Vercel account
2 Create a vercel.json file: { "version": 2, "builds": [ { "src": "*.html", "use": "@vercel/static" }, { "src": "*.js", "use": "@vercel/node" } ], "routes": [ { "src": "/api/(.*)", "dest": "api/together-api.js" } ] }
3 Create an api/together-api.js file: module.exports = async (req, res) => { const apiKey = process.env.TOGETHER_API_KEY; try { const response = await fetch("https://api.together.xyz/v1/chat/completions", { method: "POST", headers: { "Content-Type": "application/json", "Authorization": `Bearer ${apiKey}` }, body: JSON.stringify(req.body) }); const data = await response.json(); return res.json(data); } catch (error) { return res.status(500).json({ error: error.message }); } };
4 Set up environment variables in Vercel's dashboard

3. Custom Server

You can also deploy to any hosting provider that supports server-side code (Node.js, Python, etc.).

Create a simple proxy endpoint that forwards requests to the Together API with your API key.

Testing Your Deployment

After deployment, test the application by entering different situations and checking if the AI generates unique excuses each time.

Check browser console logs for any API errors.

Local Development

For local development:

npm install -g serve serve

This will start a local server at http://localhost:5000

Need Help?

If you encounter any issues with deployment, please check:

  • API key is correctly set in environment variables
  • Serverless function is properly configured
  • CORS is properly handled
